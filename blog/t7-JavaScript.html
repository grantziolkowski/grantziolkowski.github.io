Ruby Hashes vs. JavaScript Objects

Comparing data structures

February 27, 2015

Hashes in Ruby and Objects in JavaScript are similar structures used to organized data. The only differences lie in some terminology and syntax. Both are essential for grouping an object (piece of data) to one or more corresponding values. Think of a dictionary: each word is tied to its definition.

In Ruby, keys have corresponding values.

my_hash = {
apples: 2
peaches: 5
oranges: 3
}

...
end

def get_up_and_go
if @purpose == "nothing"
p "Slow down and rest at #{@location}!"
elsif @purpose == "business"
p "Get up early go to #{@location}!"
elsif @purpose == "pleasure"
p "Have fun at the #{@location}."
end
end
end

In JavaScript, properties have corresponding values.
