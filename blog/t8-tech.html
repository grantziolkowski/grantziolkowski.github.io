SQL Injection

Careful now

March 5, 2015

SQL (Structured Query Language) lets you retrieve and update data from a database.

You need to be careful when using user input for your search values on a website.

Either on purpose or by accident, a user could enter in valid SQL commands in a database field and you have potential for trouble.

Sometimes users are bad and they can inject SQL commands that can alter your SQL statement, your database, and the security of your website.

For example, imagine a website with a field asking for your user name:

User name:
" or ""="

Imagine your database read this and assigned " or ""=" to the value User.

Adding " or ""=" as the username will cause dangerous confusion when querying the UserNames database using SQL.

For example, a sample SQL query could read:

SELECT * FROM UserNames WHERE Name = '" + User + '"

Reading from the data your user entered in the User field (" or ""="), your SQL query would result in:

SELECT * FROM UserNames WHERE Name = "" or ""=""

Since WHERE ""="" is always true, THIS WOULD RETURN ALL OF THE User VALUES IN THE Usernames DATABASE.

Now imagine the field was passwords, or credit card numbers... Careful now!
